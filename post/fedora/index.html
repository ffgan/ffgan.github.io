简单记录一下fedora的使用

发表于： 0001-01-01
字数： 2273 阅读：≈ 5分钟

前言

Fedora是GNU/linux的一个发行版，背后主要是redhat，大多把Fedora看作RHEL的上游，因为redhat把许多功能添加进Fedora，稳定后再放入RHEL。因此，Fedora的更新是频繁的，适合没事就点两下更新的人群。主版本的更新周期是大概13个月。也可以采用预发布版本，但是不推荐(比如docker的仓库好像不支持预发布版本，当然可以强制安装，就是麻烦)。

安装

准备条件：

  1. 一个大一点的U盘，8G以上均可
  2. 一台电脑

下载iso镜像文件

Fedora分几种版本，一般来说开发尝鲜用Workstation版即可

https://fedoraproject.org/

官网是上面这个，找到Workstation，点Download Now，进去可以看到

先下左边这个media write，作用是把Fedora的iso文件写入U盘，可以插U盘然后装fedora到硬盘上。还需要下右边的iso镜像文件，看架构来选即可，多是x86_64.（也可以选择不下，打开media write后有自动下载的选项）

给U盘刷入系统

先确保U盘里面的文件都已经备份，在接下来的步骤里，里面的数据均会被删除 ！ ！ ！或者说将格式化整个U盘。

（如果无法启动Media Write，点击后没有反应，需要看看Windows defender的这个选项，

需要保持为默认值，如果改为打开会导致media writer起不来）

然后运行Media Writer，选好U盘，选好ISO文件，点击写入，稍微有点慢，注意这时候不要碰U盘，等待完成即可。

更改启动选项

需要稍微了解一下怎么进BIOS，这个和电脑的主板有关，有的是按delete或者F几，进BIOS，然后启动选项把U盘放到第一位。

进入之后有个选择启动的界面，默认是中间的test and start，如果期间没有触碰U盘，按向上的小方向键，然后enter回车，选第一个直接start即可。

安装系统到硬盘上

本身U盘已经蕴含了一个fedora，为live版，我们需要将这个系统再安装一份到硬盘上。

一进入Fedora里面就会提示是否安装到硬盘，右边一个就是not now，咱们选左边那个，安装。

Fedora不同于其他linux发行版，安装过程不需要科学上网，只需要一开始选一下时区、语言以及安装的目标硬盘即可，选好才能进行下一步，注意选完硬盘的确认键在左上角.耐心等待安装完成即可。

提示安装完成后，会退回到Fedora系统，点右上角的restart来重启

再进bios改启动项为硬盘，启动即可。

启动后可以拔掉U盘

启动

fedora的默认桌面是GNOME，颇具极简主义，desktop界面空白的只有壁纸，右下角的fedora的logo是可以去掉的，需要安装一个个性化插件配置

sudo dnf install gnome-tweak-tool

gnome-tweaks

这里最主要调一个缩放比例，默认fedora只支持100% 200% 300%，而我需要1.25的缩放所以需要这个插件。以及可以调出类似windows的应用右上角的最小化最大化

去掉那个logo需要安装gnome-extension。

sudo dnf install gnome-extensions-app

在app列表找到 扩展 ，里面即可去掉背景的logo

设置root密码 

sudo passwd

更新全部软件（包括内核） 

sudo dnf update -y

dnf换成yum也可以

sudo dnf upgrade -y # sudo yum upgrade -y

注意redhat系的update不同于debian的update。 debian的只是更新包的信息，并未实际更新包，upgrade才是更新软件。

neofetch 

sudo dnf install neofetch

这是一个优雅展示系统信息的软件

neofetch

会展示相关信息以及发行版的logo

安装ssh服务 

sudo dnf install openssh-server

默认应该是带上的，没有的话安一下，方便远程登录

sudo systemctl status sshd

查看ssh-server的信息，这个d后缀表示deamon，守护进程，类似Windows的服务，就是服务器程序，不带d的比如ssh是客户端。mysql、mysqld同理。

如果是disabled状态，可以使它变为开机自启

sudo systemctl enable sshd

然后启动

sudo systemctl start sshd

再次查看状态应该就是启动了，默认跑在22端口

查看网卡配置 

ifconfig

查看当前网卡信息，同时也能看到ip

咱们现在配置一下sshd

密钥登录

这部分涉及到非对称加密，贴个链接供有兴趣的查看即可

这个博主的教程主打一个事无巨细

非对称牢记一句即可，私钥不可泄露（本地ssh客户端连接需要提供私钥除外），公钥随意分发。

ssh的配置在这里

sudo vi /etc/ssh/sshd_config

密钥登陆的配置参照上面的链接即可，配置好之后就可以禁用密码登陆了

禁用密码登录 

# To disable tunneled clear text passwords, change to no here!
#PasswordAuthentication yes

把前面的#号去掉，yes改为no

禁用root登录（建议）

一般来说不允许root远程登录，除非是在docker里面为了方便进去管理

PermitRootLogin no

同上

改完之后,重启sshd

sudo systemctl restart sshd

安装chrome

安装第三方仓库

sudo dnf install fedora-workstation-repositories

启用chrome的仓库

sudo dnf config-manager --set-enabled google-chrome

安装chrome稳定版

sudo dnf install google-chrome-stable

安装vscode

同样，先导入源

sudo rpm --import https://packages.microsoft.com/keys/microsoft.asc
sudo sh -c 'echo -e "[code]\nname=Visual Studio Code\nbaseurl=https://packages.microsoft.com/yumrepos/vscode\nenabled=1\ngpgcheck=1\ngpgkey=https://packages.microsoft.com/keys/microsoft.asc" > /etc/yum.repos.d/vscode.repo'

然后就可以安装了，如果下面这步没装上，上面的命令多执行几次，网络问题

dnf check-update
sudo dnf install code # or code-insiders

去掉 您登陆计算机时，未解锁密钥环 的提示 

rm -r ~/.local/share/keyrings/login.keyring

然后重启，再打开chrome/vscode，要求输入密码的时候直接为空即可

Golang/Go

Go是编译型语言，所以需要安装编译器来编译运行

sudo dnf install golang

go version

go version go1.21.7 linux/amd64
