本文假定阅读对象为刚入门的c/c++新手,无前置要求

开始

1. 了解c语言的hello world

以下给出一个实例

#include <stdio.h>
int main()
{
    printf("hello world!");
    return 0;
}

以上是一段c语言的入门代码。

编程语言发展经过了三代，一是可直接执行的机器语言，二是汇编语言，三是高级语言。

这段代码对于cpu来说仅是一段普通的文本，没有任何意义。

所以我们如果希望执行以上这段代码，就需要将代码编译、链接然后生成可执行文件，简而言之在win下就是将代码变成可执行的二进制文件，如exe文件（executable file）。

没有魔法能直接念咒然后将代码生成你想要的二进制文件，然后执行显示hello world。所以我们需要一个可以将代码转换成二进制文件的软件。

编译器：正是在下

对于c/c++来说，最出名的编译器莫过于GNU（GNU NOT UNIX，取首字母）项目的gcc/g++，开源的自由软件，不过这个软件和平常用的通过点击或者触控来操作的（GUI， Graphical User Interface）图形化软件不同，这是命令行软件（CLI，Command Line Interface），意即它没有操作界面，只有黑乎乎的界面，有点像黑客那样。

所以咱们本次主要需要下载两个软件，一个是vscode（Visual Studio Code，微软家的一个文本编辑器，和记事本本质没区别，都是打开文本文件然后修改里面的字符串，但是vscode添加了插件，支持的编程语言以及集成了部分便于写程序的功能），有了vscode我们就能编写代码。二是gcc编译器，写完代码我们就利用gcc将其编译为二进制可执行文件。

PATH环境变量

PATH是类Unix系统、DOS、OS/2和Microsoft Windows操作系统上的一个环境变量，用于设置一组包含可执行文件的目录。

维基百科copy过来的。

先介绍啥是环境变量

这个可以理解为一堆从开机开始就一直存在的变量，值是预先设定的，比如我设定了一个环境变量a，值为10。那么我在shell就可以读取并且使用这个变量。有什么用呢，可以理解为一个配置文件，有些应用程序需要使用到这些环境变量。比如我们就希望能直接运行a.exe而不是记一堆路径。方法呢也很简单，但是介绍cli的方式来设置环境变量知识点容易过载而且不利于理解网上的关于win下设置环境变量的做法。

所以如果说，我们预先把a.exe塞到PATH环境变量里那么我们就能直接运行它，因为cmd在你输入一个命令的时候会到看看这是不是一个路径，是路径就按照路径给出的去找，找不到就再往PATH里查，还找不到怎么办，报错¬_¬，说找不到。

{

    "version": "0.2.0",
    "configurations": [
        {
            "name": "(gdb) 启动",
            "type": "cppdbg",
            "request": "launch",
            "program": "${fileDirname}\\${fileBasenameNoExtension}.exe",
            "args": [],
            "stopAtEntry": false,
            "cwd": "${fileDirname}",
            "environment": [],
            "externalConsole": false,
            "MIMode": "gdb",
            "miDebuggerPath": "gdb",
            "setupCommands": [
                {
                    "description": "为 gdb 启用整齐打印",
                    "text": "-enable-pretty-printing",

                },
                {
                    "description": "将反汇编风格设置为 Intel",

                    "ignoreFailures": true
                }
            ],
            "preLaunchTask": "C/C++: gcc.exe 生成活动文件"
        }

    ]
}

这个prelaunch很关键，回收之前讲的，按下f5之后，会执行launch.json配置的命令行，然后这个preLaunchTask意思是在执行launch之前先运行一个task，后面的参数指明运行哪个，刚好来自于tasks.json里的label里的值

我再给出我的tasks.json

{
    "tasks": [
        {
            "type": "cppbuild",
            "label": "C/C++: gcc.exe 生成活动文件",
            "command": "D:\\devTools\\w64devkit\\bin\\gcc.exe",
            "args": [
                "-fdiagnostics-color=always",
                "-g",
                "${file}",
                "-o",
                "${fileDirname}\\${fileBasenameNoExtension}.exe"
            ],
            "options": {
                "cwd": "${fileDirname}"
            },
            "problemMatcher": [
                "$gcc"
            ],
            "group": {
                "kind": "build",
                "isDefault": true
            },
            "detail": "调试器生成的任务。"
        }
    ],
    "version": "2.0.0"
}

完结。 接下来就可以随意的按f5就会重新编译然后运行该文件

总结

综上来看，我们配置的关键在于

第一，指定launch.json需要运行的程序，然后修改gdb的路径，为什么我们可以直接写gdb（因为我们添加的bin文件夹里有它）。然后添加一项preLaunchTask，指定我们希望在启动该程序前先重新编译一次，它的值来自tasks.json里的label的值。

第二，tasks.json里面实际上就是在配置如何进行编译，里面那些参数就是传递给编译器的参数，我们也可以修改为启动一个shell脚本执行特殊或者复杂的编译操作

这里的gcc路径也可以直接改为

gcc

直接使用名字就行,和直接使用gdb有异曲同工之妙，然后就顺利配置完成了。
